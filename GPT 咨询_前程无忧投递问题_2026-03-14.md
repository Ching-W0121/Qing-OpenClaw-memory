## 🎯 核心原因：ref 失效了

**问题本质**: 前程无忧 (51job) 是 **React SPA** 单页应用，点击职位后 DOM 会重新渲染。

```
snapshot → ref(e190) → DOM index pointer → 只在当前 DOM 树有效
                        ↓
              点击职位后 DOM 重新渲染
                        ↓
              e190 → 指向不存在的节点 → 点击失败
```

### 为什么第 1 个成功？
- 初始 snapshot 后，DOM 还没变化
- 流程：`snapshot → click job → click apply` 还能用

### 为什么第 2-5 个失败？
- 点击第 1 个职位后，列表 DOM 重新渲染
- 原来的 `e190`、`e280` 已经失效
- 点击的是 **错误的 DOM 节点**

---

## ✅ 正确的自动化流程

### 关键规则（必须遵守）

| 规则 | 说明 |
|------|------|
| **规则 1** | 任何点击后都要重新 snapshot |
| **规则 2** | 绝对不要复用旧 ref |
| **规则 3** | 列表页必须重新 snapshot |

### 正确流程

```
1. 打开列表页
2. snapshot → 获取职位 ref (e101, e102, e103)
3. click e101 (点击职位)
4. wait 3000-5000ms (等待加载)
5. snapshot → 重新获取投递按钮 ref (e512)
6. click e512 (点击投递)
7. snapshot → 确认投递状态
8. browser.back() 或 browser.open(list_url) 返回列表
9. snapshot → 重新获取下一个职位 ref
10. 重复步骤 3-9
```

---

## 🛡️ 成功率优化建议

### 1. 加随机等待
- 每次操作之间等待 **2-5 秒**
- 避免触发反自动化检测

### 2. 滚动页面
- 有些职位的投递按钮在 viewport 外
- 需要先滚动到可见区域

### 3. 每投递 3 个暂停
- `sleep 20s`
- 模拟人类行为，降低封禁风险

### 4. 判断投递按钮状态
前程无忧投递按钮有 3 种状态：
- `投递`
- `立即投递`
- `已投递`

必须判断按钮文本，避免重复投递失败。
