## 五、API 架构

### 5.1 端点层次

```
求职 Agent API v1.2.0
├── /                      # 根路径
├── /health                # 健康检查
├── /docs                  # API 文档（Swagger）
│
├── /api/users             # 用户管理
│   ├── POST   /          # 创建用户
│   └── GET    /{id}      # 获取用户
│
├── /api/jobs              # 职位管理
│   ├── POST   /search    # 搜索职位
│   └── GET    /{id}      # 获取详情
│
├── /api/matches           # 匹配推荐
│   ├── POST   /calculate # 计算匹配
│   └── GET    /recommend/{id}  # 获取推荐
│
└── /api/applications      # 投递管理
    ├── POST   /submit    # 提交投递
    └── GET    /{id}      # 获取记录
```

### 5.2 数据模型

```python
UserProfile:

  - skills: List[str]
  - tools: List[str]
  - experience_years: int
  - education: str
  - expected_city: str
  - expected_salary_min/max: int

Job:
  - id: str
  - title: str
  - company: str
  - industry: str
  - skills: List[str]
  - tools: List[str]
  - experience: str
  - education: str
  - city: str
  - salary_min/max: int

Match:
  - job_id: str
  - match_rate: float (0-100)
  - level: str (极佳/很好/一般/低)
  - details: dict
```
