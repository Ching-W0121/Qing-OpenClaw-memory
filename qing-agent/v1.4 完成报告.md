## 🐛 遇到的问题及解决

### 问题 1: platform 目录冲突

**现象**: SQLAlchemy 导入失败，报错 `module 'platform' has no attribute 'python_implementation'`

**根因**: 项目目录 `platform/` 与 Python 标准库 `platform` 模块冲突

**解决**: 重命名 `platform/` → `qing_platform/`

**教训**: 避免使用 Python 标准库名称作为目录名

---

### 问题 2: FastAPI response_model 类型错误

**现象**: `FastAPIError: Invalid args for response field`

**根因**: 使用 SQLAlchemy 模型作为 response_model，FastAPI 需要 Pydantic 模型

**解决**: 创建 `schemas.py`，使用 Pydantic v2 的 `from_attributes=True`

**教训**: FastAPI 响应模型必须使用 Pydantic，不能用 SQLAlchemy

---

### 问题 3: SQLAlchemy 版本兼容性

**现象**: 导入时出现 `AttributeError: module 'platform' has no attribute...`

**根因**: SQLAlchemy 2.0.48 与 Python 3.11.9 兼容性问题

**解决**: 降级到 SQLAlchemy 2.0.36

**教训**: 生产环境使用稳定版本，不盲目追求最新版
