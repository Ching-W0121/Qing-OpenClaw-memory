## 🐛 重要问题与解决

### 1. 死循环问题（重要教训）
**现象**: 反复尝试打开同一个打不开的 ChatGPT 链接

**原因**:
- 过度执着于单一路径
- 缺乏灵活的 fallback 机制
- 反思系统没有真正生效
- 工具使用僵化

**解决**:
- 在 AGENTS.md 中添加"3 次法则"
- 失败 3 次必须停下来换方法
- 真正的 resourceful 是懂得灵活切换策略

**记忆点**: 当同一个操作失败 3 次时，必须停下来换方法，而不是继续尝试。

### 2. BOSS 直聘网页端不可用
**现象**: 访问 zhipin.com 后被重定向到智联招聘

**原因**: BOSS 直聘有严格的反自动化检测（国内最严格之一）

**决策**: 按 GPT 老师建议，第一阶段不死磕 BOSS，用智联招聘作为主数据源

### 3. 反思系统不生效
**原因**: 技能文件缺失（显示 ready 但实际文件不存在于磁盘）

**解决**: 在 AGENTS.md 中添加强制反思流程（双重反思机制）

## 💡 关键学习

### 技术层面
1. 导入路径错误是常见问题（platform → qing_platform）
2. 保护机制必须包含：重试 + 超时 + 熔断器 + 降级
3. Pipeline 设计要模块化，便于扩展
4. SQLite 适合开发，生产环境需要 PostgreSQL

### 工程层面
1. 不要死磕一个问题，学会切换策略
2. 稳定数据源比数据量更重要
3. 逐步升级比推翻重来更明智
4. 用户反馈是优化推荐的关键

### 产品层面
1. v2.0 核心不是功能堆砌，而是让用户更容易找到好工作
2. 关键指标：CTR、投递转化率、面试率、offer 率
3. 个性化推送比固定时间推送更有效
