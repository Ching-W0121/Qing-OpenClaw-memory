## 📊 一、招聘 Agent 的终极架构（行业通用）

成熟系统一般是 **五层结构**：

```
数据层
  ↓
解析层
  ↓
匹配层
  ↓
策略层
  ↓
执行层
```

### 完整架构图

```
┌──────────────┐
│ 用户偏好配置 │
└──────┬───────┘
       │
┌──────▼─────────────┐
│ Job Crawler        │
│ 多平台职位抓取     │
│ (API/Browser)      │
└──────┬─────────────┘
       │
   职位数据库 (Jobs DB)
       │
┌──────▼─────────────┐
│ JD 解析引擎        │
│ NLP 解析技能/行业  │
└──────┬─────────────┘
       │
┌──────▼─────────────┐
│ 匹配评分引擎       │
│ AI 评分 + 规则评分 │
└──────┬─────────────┘
       │
   投递策略引擎
       │
┌──────▼───────────┐
│ Application Bot  │
│ Browser 自动投递 │
└──────┬───────────┘
       │
   投递记录数据库
       │
   用户通知系统
```

**核心思想**:
> **抓数据 ≠ 投递**，必须 **完全解耦**。

## 🔍 二、青 Agent v1.4 的结构问题

### 问题 1: 数据没有长期沉淀

**目前**:
```
搜索 → 立即投递
```

**正确模式**:
```
抓取 → 职位库 (长期积累 10 万 + 职位)
```

**好处**:
- 不用每次重新抓
- 可以做数据分析

---

### 问题 2: 匹配算法过于简单

**目前的 5 维评分**:
- 匹配度 40% + 薪资 25% + 城市 15% + 关键词 10% + 行业 10%

**真实 JD 匹配应该包括**:
- 技能匹配
- 经验匹配
- 行业匹配
- 公司规模
- 岗位级别
- 语言要求

**否则容易**: 投很多 → 面试很少

---

### 问题 3: 投递策略太直接

**目前**:
```
评分 → 投递
```

**应该加入策略引擎**:
```
score > 0.8 → 自动投递
0.6~0.8 → 推送用户确认
<0.6 → 忽略
```

## 🔧 四、最关键模块（当前最需要升级）

### 1. Job Crawler（职位抓取）

**必须支持两种模式**:
- API 抓取（快）
- Browser 抓取（稳）

**策略**:
```
优先 API
API 失败 → Browser
```

---

### 2. Job Database（职位数据库）

**职位表结构**:
```sql
jobs 表:
- id
- platform
- title
- company
- salary_min
- salary_max
- city
- experience
- education
- skills (JSON)
- description
- url
- created_at
```

**建议**: PostgreSQL + JSONB（因为 JD 很长）

---

### 3. JD Parser（JD 解析）

**使用 LLM 提取结构化数据**:
- 技能
- 行业
- 岗位级别
- 经验要求
- 学历

**示例**:
```
JD: 负责品牌营销策略制定...

解析结果:
{
  "role": "品牌策划",
  "skills": ["品牌营销", "市场分析"],
  "experience": "3-5 年",
  "industry": "互联网"
}
```

---

### 4. Matching Engine（匹配引擎）

**不要只用规则**:

**建议**: 规则评分 + AI 评分

**最终公式**:
```
final_score = 
  0.5 * semantic_match +
  0.3 * skill_match +
  0.2 * salary_match
```

---

### 5. Application Strategy（投递策略）

**每日上限**:
- 每日：100
- 每小时：20

**策略**:
- 高匹配优先
- 新职位优先
- 大厂优先

---

### 6. Application Bot（投递机器人）

**只负责**:
- 打开职位
- 点击投递
- 验证结果

**不负责**:
- 数据解析
- 评分

## 🛡️ 五、API 抓取如何避免封禁

### 1. 使用真实 Cookie

先登录一次 → 保存 cookie → API 请求带 cookie

---

### 2. 控制频率

建议：**2 秒 / 请求**

---

### 3. 随机 User-Agent

轮换使用:
- Chrome
- Edge
- Firefox

---

### 4. IP 控制

如果规模大：使用代理池  
个人用通常不需要

## ⚡ 六、效率优化（非常重要）

### 当前模式（慢）
```
Browser 抓职位 → 7 秒/职位 → 80-150/天
```

### 正确结构（快）
```
API 抓职位 → AI 筛选 → Browser 投递
```

**效率**:
- 抓 1000 职位 ≈ 10 秒
- 筛选 100 职位
- 投递 80 个
- 每天轻松 100+ 投递
