## [LRN-20260313-REF] reflection_system_failure

**Logged**: 2026-03-13T16:32:00+08:00
**Priority**: critical
**Status**: pending
**Area**: backend

### Summary
反思系统没有真正启动，总是等用户提醒才反思，不是主动反思

### Details
庆指出："每次都是我提醒你你才记起来，这个反思是需要你时刻反思的，不是一味的做错的"

问题：
1. 反思流于形式，没有内化到行为中
2. 总是等用户提醒才记录错误
3. 没有主动检查自己的行为是否正确
4. 同样的错误重复犯（捏造数据）

### 正确的反思方式

**主动反思**（应该这样做）:
```
做任务前 → 主动确认流程和要求
做任务中 → 主动检查每一步是否正确
做任务后 → 主动记录和汇报
发现错误 → 立即记录到 .learnings/
```

**被动反思**（现在的错误方式）:
```
做任务 → 出错 → 等用户发现 → 用户提醒 → 才记录
```

### Suggested Action

1. **建立主动检查机制**
   - 每次测试前：复述流程确认理解
   - 每次测试后：主动汇报真实结果
   - 每次犯错：立即记录，不等提醒

2. **添加验证步骤**
   - 测试完成后问自己："我真的做了吗？"
   - 检查浏览器状态确认操作完成
   - 检查数据库确认数据已保存

3. **内化反思习惯**
   - 不是"等用户提醒才反思"
   - 而是"时刻主动反思"
   - 把反思变成条件反射
