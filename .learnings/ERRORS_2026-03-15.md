## [ERR-20260315-002] 不检索记忆就盲目操作 🔴

**Logged**: 2026-03-15T01:30:00+08:00
**Priority**: critical
**Status**: pending
**Area**: workflow

### Summary
Gemini 咨询失败后，没有检索记忆中下午成功的记录，盲目重复操作导致死循环

### Details
**问题经过**：
1. 15:22 - 下午成功咨询 Gemini 老师（职位数据库设计）
2. 01:23 - 晚上再次咨询 Gemini（FAISS 架构问题）
3. 失败后没有检索记忆查看下午的成功方法
4. 陷入等待死循环
5. 庆指出后才检查记忆

**根本原因**：
- 没有养成"失败后检索记忆"的习惯
- 没有反思系统自动触发
- 重复同样的错误

**下午成功的方法**：
```
1. browser.open("https://gemini.google.com")
2. snapshot(delay=5000)  # 等页面加载
3. 找到输入框（e4 或 e221）
4. type + press Enter
5. snapshot(delay=30000)  # 等回复
```

**正确做法**：
```
失败 → memory_search("Gemini 咨询") → 查看成功记录 → 复制方法
```

### Suggested Action
1. 失败后强制检索记忆
2. 建立"成功模式"库
3. 优先复制成功方法
