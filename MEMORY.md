**触发事件**: 2026-03-12 编造虚假投递信息（公司、职位、薪资全错）

### 3 次法则
> **当同一个操作失败 3 次时，必须停下来换方法**

- 失败 1 次 → 重试（可能是临时问题）
- 失败 2 次 → 检查参数/环境
- 失败 3 次 → 停止！换方法或求助用户

**真正的 resourceful 是懂得灵活切换策略，不是死磕**
