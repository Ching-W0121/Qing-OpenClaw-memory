### 3 次法则
> **当同一个操作失败 3 次时，必须停下来换方法**

- 失败 1 次 → 重试（可能是临时问题）
- 失败 2 次 → 检查参数/环境
- 失败 3 次 → 停止！换方法或求助用户

**真正的 resourceful 是懂得灵活切换策略，不是死磕**

## 🚨 重要教训记录

### 5. 动态网页元素失效（2026-03-14）🔴
**问题**: GPT 页面发送消息时，snapshot 后等待太久导致 ref 失效，重复 10+ 次
**原因**: 
- React 应用持续重新渲染
- snapshot 只捕获瞬间 DOM
- 等待后 ref 失效还用旧元素操作
- 不验证结果就假设成功
**解决**: 
- 创建强制执行系统（memory_guard.py）
- 失败计数器（3 次强制停止）
- 操作前强制检索记忆
- 操作后必须验证结果

- 创建 `memory/failure_tracker.md` - 失败计数器

### 4. 记忆系统数据丢失（2026-03-13）🔴
**问题**: 用户手动在电脑端完成智联投递测试后，多次要求获取投递数据，但系统无记录
**原因**: 
- 用户手动在浏览器/APP 操作投递，系统无自动记录
- 对话中提到的公司信息未被 episodic memory 捕获
**解决**: 
- 立即：向用户承认数据丢失，请求重新提供
- 短期：实现投递数据手动录入功能
- 长期：开发浏览器插件自动捕获投递数据

### 5. 语义检索不完整导致用户重复询问 5 次（2026-03-13）🔴
**问题**: 用户多次询问智联投递数据，我检查了数据库和记忆文件后说数据丢失，但实际上数据一直在 `智联招聘_品牌策划_2026-03-13.md` 文件中
**原因**: 
- 只检查了数据库和 memory/ 目录，没有搜索 workspace 根目录的 markdown 文件
- 思维定式：认为数据"应该在数据库中"
- 没有遵守 3 次法则：用户问了 3 次后应该换方法
**教训**: 搜索范围必须包括 workspace 根目录所有文件

### 6. 前程无忧自动化测试部分失败（2026-03-13）
**问题**: 5 个职位仅成功 1 个（冠旭电子），其余 4 个失败
