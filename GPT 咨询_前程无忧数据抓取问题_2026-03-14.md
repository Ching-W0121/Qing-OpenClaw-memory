## 🎯 核心问题

**问题**：前程无忧和智联招聘都使用 `browser.snapshot() + browser.act()` 进行自动化投递，但结果不同：
- **智联招聘**：能成功返回真实的职位数据（公司、职位、薪资等）
- **前程无忧**：投递成功后，无法抓取到真实的职位数据

---

## 💡 GPT 核心结论

> **前程无忧的问题不是抓不到，而是你抓的时机错了。**

> **智联招聘和前程无忧的页面架构完全不同，所以你的流程在智联能跑，在前程无忧就会失效。**

---

## 📊 一、核心差异：智联 vs 前程无忧

### 1️⃣ 智联招聘

**流程**：
```
职位列表页
  ↓
点击职位
  ↓
职位详情页 (完整 DOM)
  ↓
点击投递
  ↓
投递成功提示 (但职位 DOM 不变)
```

**关键特点**：
- **投递前后 DOM 基本不变**
- 因此：`snapshot → act → snapshot` 还能抓到公司、职位、薪资

---

### 2️⃣ 前程无忧（关键区别）

**流程**：
```
职位详情页
  ↓
点击投递
  ↓
弹窗 / overlay
  ↓
投递成功 toast
```

**问题在这里**：
- **投递成功后，页面 focus 会切到 overlay**
- `browser.snapshot(refs="aria")` 通常只抓**当前 active accessibility tree**

**结果**：
```
投递前 snapshot → 有职位信息
投递后 snapshot → 只剩 overlay（职位信息丢失）
```

## ✅ 三、正确工程方案（非常重要）

**不要**：投递后抓数据  
**而是**：**投递前抓数据**

**标准流程**：
```
打开职位
  ↓
snapshot
  ↓
解析职位数据
  ↓
保存数据库
  ↓
点击投递
  ↓
验证投递状态
```

## 📝 四、推荐代码结构

```python
async def _apply_51job(self, job, resume=None):
    # 1 打开职位
    await browser.open(job.get('url'))
    await asyncio.sleep(random.uniform(3, 5))
    
    # 2 先 snapshot 抓职位数据
    snapshot = await browser.snapshot(refs="aria")
    job_data = self._parse_job_info(snapshot)
    
    # 3 找投递按钮
    apply_ref = self._find_apply_button_ref(snapshot)
    
    # 4 点击投递
    await browser.act(kind="click", ref=apply_ref)
    await asyncio.sleep(random.uniform(2, 3))
    
    # 5 检查投递状态
    result_snapshot = await browser.snapshot(refs="aria")
    snapshot_text = str(result_snapshot).lower()
    
    if "投递成功" in snapshot_text:
        return {
            "status": "success",
            "job": job_data
        }
    
    return {
        "status": "failed",
        "job": job_data
    }
```

## 🔧 五、职位信息解析函数

```python
def _parse_job_info(self, snapshot):
    text = str(snapshot)
    job = {}
    
    # 解析职位名
    title = re.search(r'generic "(.+? 岗)"', text)
    # 解析薪资
    salary = re.search(r'generic "(\d+-\d+.*? 薪)"', text)
    # 解析公司名
    company = re.search(r'link "(.+?)"', text)
    
    if title:
        job["title"] = title.group(1)
    if salary:
        job["salary"] = salary.group(1)
    if company:
        job["company"] = company.group(1)
    
    return job
```

## 🚀 六、更高级的做法（强烈推荐）

**GPT 建议**：其实你现在的方法还是**不够稳定**。

**真正做招聘 Agent 的工程师会**：
> **列表页抓数据，而不是详情页。**

**原因**：
```
列表页 = 稳定
详情页 = 动态
```

**推荐架构**：
```
搜索页
  ↓
snapshot
  ↓
解析职位卡片
  ↓
保存职位信息
  ↓
进入详情页
  ↓
执行投递
```

**数据库结构**：
```sql
jobs 表：
- id
- title
- company
- salary
- city
- url
- platform

applications 表：
- job_id
- apply_time
- status
```

**这样**：投递只是更新 `applications` 表，而不是重新抓数据。

## ⚠️ 七、前程无忧的隐藏坑

### 坑 1：iframe

前程无忧很多职位详情在 **iframe** 中，snapshot 有时候抓不到。

**解决**：
```python
browser.open(url)
browser.wait_for_iframe()
snapshot
```

---

### 坑 2：React hydration

有些职位信息是 **JS 渲染**，sleep 3 秒有时不够。

**建议**：
```python
wait_for_selector(".job-title")  # 等待关键元素出现
```

---

### 坑 3：反自动化

前程无忧检测：
- 操作频率
- 点击轨迹

**建议**：
```python
# 随机 4-8 秒
await asyncio.sleep(random.uniform(4, 8))
```
